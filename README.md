# aggregator

`aggregator` 是一个面向高端自媒体创作者、MCN 和品牌内容团队的内容创作平台原型项目。

它的目标不是先做一个完整后端系统，而是先用高保真前端 Demo 把产品体验跑通：让客户能直观看到「热点洞察、爆款拆解、AI 创作、素材管理、内容日历、发布中心、数据看板、团队协作」这一整条内容生产线。

## 这个项目解决什么问题

- 帮内容团队从「单点 AI 工具」升级成「内容生产闭环」。
- 用数据驱动选题、创作、发布和复盘，减少靠感觉做内容。
- 给销售或客户沟通一个可点击、可演示的产品原型。
- 在真实接 API、AI、多平台发布之前，先验证客户是否认可产品方向。

## 当前形态

- `prototype/`：主要前端原型页面，包含首页、洞察、分析、创作、素材、日历、发布、团队、价格等页面。
- `doc/`：产品方案、报价、风险分析、功能完善建议等文档。
- 当前重点是产品验证和方案沟通，不是生产级 SaaS 后端。

## 核心页面

- `prototype/index.html`：产品首页和价值主张。
- `prototype/dashboard.html`：内容数据看板。
- `prototype/insight.html`：热点洞察。
- `prototype/analyze.html`：爆款拆解。
- `prototype/create.html`：多平台内容创作。
- `prototype/publish.html`：发布中心。
- `prototype/team.html`：团队协作。
- `prototype/pricing.html`：价格方案。

## 技术与设计

- 主要是静态 HTML/CSS/JS 原型。
- 设计方向偏高端内容工具，使用深色、暖金、克制的产品界面。
- 数据以 mock 为主，后续可接新榜、蝉妈妈、灰豚等内容数据 API。
- AI 能力规划走 `sub2api` 统一中转，避免前期绑定单一模型供应商。

## 后续建议

1. 先整理 Demo 演示路径，保证客户 3 分钟内理解产品价值。
2. 再确定 P0 功能：热点洞察、爆款拆解、文案创作、数据看板。
3. 最后再接真实数据源、账号授权和多平台发布能力。
